# Rust 기반 테스트 검증을 위한 보드게임 프로젝트


## 🎯 프로젝트 개요
본 프로젝트는 **Rust 언어 기반에서의 테스트 가능성 및 구조 설계 검증**을 목표로 진행되는 실험적 프로젝트입니다.  
이를 위해 명확한 규칙과 상태 변화를 갖는 부루마불 형태의 보드게임을 구현하였으며,  
게임 로직을 테스트 가능한 구조로 설계하는 데 초점을 맞추었습니다.  


## 🧩 Main 기능
- 주사위 랜덤값 생성
- 플레이어 이동 처리
- 턴 기반 게임 진행
- 자산 시스템: 토지 구매 및 통행료 지불
- 이벤트 시스템: 월급 수령, 세금 납부 등
- 게임 종료 및 최종 순위 산정

각 기능은 독립적으로 테스트 가능하도록 스크립트를 분리하여 구현하였습니다.


## 🗂️ 프로젝트 구조
```
├─ assets: UI에 사용되는 이미지
├─ description: 프로젝트와 관련된 문서
├─ project: 리팩토링 전 백엔드 코드 (현재는 사용하지 않음)
│  ├─ src
|  |  ├─ init_db: DB 초기화 스크립트
|  |  ├─ dto.rs: 구조체 정의 코드
|  |  ├─ handler.rs: API 구현 코드
|  |  ├─ main.rs: 백엔드 파이프라인
|  |  └─ service.rs: 게임 로직
│  ├─ target: 백엔드 실행 후 산출물
│  └─ Cargo.toml: 프로젝트 정보 저장소
├─ project_seperate
│  ├─ src
|  |  ├─ handler: 프론트-백 API 연동 스크립트
|  |  ├─ repository: DB 생성, 갱신, 상태관리 스크립트
|  |  ├─ service: 게임 로직
|  |  ├─ lib.rs
|  |  └─ main.rs: 백엔드 파이프라인
│  ├─ tests: 테스트 코드
│  ├─ target: 백엔드 실행 후 산출물
│  └─ Cargo.toml: 프로젝트 정보 저장소
├─ index.html
├─ result.html
├─ map.js
└─ style.css
```
### Frontend (HTML, CSS, JS)
- 사용자 인터페이스
- 화면 설계 및 디자인
- 게임 흐름 시각화
- API 연동
### Backend (Rust)
- DB 설계
- 게임 로직 처리
- 상태 관리
- 테스트 코드 작성


## 👀 테스트 관점에서의 포인트
- 상태 기반 로직(Stateful Logic)의 테스트 가능 여부
- 조건 분기(if, &&, ||)에 대한 테스트 커버리지 확보
- 입력 → 상태 변화 → 결과 검증 구조 확립
- 서비스 레이어 단위 테스트 가능성
